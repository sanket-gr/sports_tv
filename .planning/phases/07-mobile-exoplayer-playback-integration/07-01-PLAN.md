# Plan 07-01: Mobile ExoPlayer Playback Integration

Bootstrap HLS video playback for the mobile application module, providing a responsive touch interface with customized controls, automatic headers, and robust orientation handling.

## User Review Required

> [!IMPORTANT]
> The playback orientation will support both portrait and landscape. Rotating to landscape will hide system bars automatically to enter full-screen mode, while rotating back to portrait restores normal view. The MainActivity remains locked to portrait as required.

## Open Questions

None.

## Proposed Changes

### Mobile App Module

---

#### [MODIFY] [AndroidManifest.xml](file:///d:/projects/sports_tv/android_tv_app/mobile/src/main/AndroidManifest.xml)
- Declare `PlaybackActivity` with `configChanges` set to `orientation|screenSize|keyboardHidden|screenLayout|smallestScreenSize` so rotation is handled dynamically in code without recreating the activity.
- Add hardware acceleration support if not already present.

#### [NEW] [activity_playback.xml](file:///d:/projects/sports_tv/android_tv_app/mobile/src/main/res/layout/activity_playback.xml)
- FrameLayout root holding the `androidx.media3.ui.PlayerView` matching parent.
- Overlay controls container (auto-hides after delay):
  - Top header: Back button (arrow) and title text.
  - Center: Large Play/Pause overlay button.
  - Bottom: Aspect Ratio toggle button and Volume control slider (SeekBar).
- Loading/Buffering overlay with progress bar.
- Error state overlay with Retry button.

#### [NEW] [PlaybackActivity.kt](file:///d:/projects/sports_tv/android_tv_app/mobile/src/main/java/com/sportstv/mobile/PlaybackActivity.kt)
- Create `PlaybackActivity` subclassing `AppCompatActivity`.
- Initialize `ExoPlayer` using a cloned version of the TV app's `DynamicHeaderDataSource` to append correct headers (`Referer`, `Origin`, etc.) for sportsurge and suisports streams.
- Implement touch listeners to toggle controls visibility (fade-in on tap, fade-out after 3.5 seconds of inactivity).
- Hook controls:
  - Play/Pause click listener toggling player state.
  - SeekBar listener maps to `player.volume` (0.0f to 1.0f).
  - Screen ratio toggle cycles `playerView.resizeMode` through `RESIZE_MODE_FIT`, `RESIZE_MODE_FILL`, and `RESIZE_MODE_ZOOM`.
- Override `onConfigurationChanged` to dynamically toggle full screen:
  - In landscape: Hide system status/navigation bars, set player view to fill screen.
  - In portrait: Show system status/navigation bars, restore constraints.
- Implement lifecycle hooks (pause on pause/stop, release on destroy).

#### [NEW] [item_stream_card.xml](file:///d:/projects/sports_tv/android_tv_app/mobile/src/main/res/layout/item_stream_card.xml)
- A clean Material card layout for displaying stream cards in a list (title, category badge, and thumbnail icon/text).

#### [NEW] [StreamAdapter.kt](file:///d:/projects/sports_tv/android_tv_app/mobile/src/main/java/com/sportstv/mobile/StreamAdapter.kt)
- Standard `RecyclerView.Adapter` that displays a list of `StreamItem`s, supporting touch clicks to trigger an action callback.

#### [MODIFY] [MainActivity.kt](file:///d:/projects/sports_tv/android_tv_app/mobile/src/main/java/com/sportstv/mobile/MainActivity.kt)
- Replace placeholder TextView inside FrameLayout container with a `RecyclerView`.
- Bind `StreamAdapter` to the fetched streams list.
- On stream item click, launch `PlaybackActivity` passing the selected `StreamItem` data as Intent extras.
